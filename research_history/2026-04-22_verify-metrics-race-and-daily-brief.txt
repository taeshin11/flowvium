  - 즉, getBaseUrl(req)이 외부에서 오는 alias host를 정상 반환하더라도 그 alias가 가리키는 edge function이 stale deployment일 수 있음.

**수정 (src/app/api/cron/verify-metrics/route.ts)**:
verifyEarnings을 env-first로 변경:
1. `process.env.FINNHUB_KEY` 직접 확인 → 없으면 바로 degraded 리턴
2. 있으면 /api/earnings fetch 후 응답의 warning은 'endpoint stale'로 분류 (error 아님)
→ 이제 키가 실제로 설정된 경우 verify-metrics가 false-negative 안 냄.

### 2. /api/flow-analysis 503 "capital-flows data unavailable"
**원인**: `process.env.VERCEL_URL` 사용
```
const baseUrl = process.env.VERCEL_URL
  ? `https://${process.env.VERCEL_URL}`
  : 'http://localhost:3000';
```
VERCEL_URL = 배포 specific URL (예: flowvium-of3g8t11x-...vercel.app) → **Vercel deployment protection 때문에 401** → JSON 파싱 실패 → capitalData=null → 503.

**수정 (src/app/api/flow-analysis/route.ts)**:
```
const reqHost = new URL(request.url).host;
const reqProto = new URL(request.url).protocol;
const baseUrl = reqHost.startsWith('localhost')
  ? 'http://localhost:3000'
  : `${reqProto}//${reqHost}`;
```
request.url.host = public alias (flowvium.vercel.app). 내부 fetch가 auth-protected deployment URL 안 건드림.
