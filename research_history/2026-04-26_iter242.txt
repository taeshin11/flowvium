### Bug: economic-calendar FOMC actual 영구 null

**문제:**
`/api/economic-calendar` 의 `FOMC_2026` 정적 배열에서
`actual` 필드가 모든 회의에 대해 `null` 하드코딩됨.
과거 회의(Jan 29, Mar 19)도 "Actual: N/A" 표시.

**근본 원인:**
`injectFomcEvents()` 가 항상 `actual: null` 주입.
`FOMC_2026` 타입에 `actual` 필드가 없어 값을 전달할 방법 없음.

**실제 값:**
- Jan 29, 2026: hold at 4.25% (확인: March 19 `prev: 4.25` = January 이후 변동 없음)
- Mar 19, 2026: 50bp cut to 3.75% (확인: FRED DFEDTARU=3.75 — iter241 verified)
  - estimate=4.00 (25bp cut 예상) vs actual=3.75 (50bp cut 실행) = 큰 surprise

**사용자 영향 (수정 전):**
- 경제 캘린더 1월/3월 FOMC 항목: "Actual: N/A"
- estimate=4.00 but actual=N/A → 회의 결과 미확인처럼 보임

**수정 (commit 5f5eda2):**
```typescript
// FOMC_2026 타입에 optional actual 필드 추가
type: Array<{ date: string; time: string; prev: number; estimate: number; actual?: number }>

// 과거 회의 actual 값 설정
{ date: '2026-01-29', ..., actual: 4.25 },  // hold confirmed
{ date: '2026-03-19', ..., actual: 3.75 },  // 50bp cut (FRED DFEDTARU=3.75)

// injectFomcEvents: null 하드코딩 → 전달값 사용
actual: f.actual ?? null,  // was: actual: null
```

**사용자 영향 (수정 후):**
- Jan 29 FOMC: "Estimate 4.25% | Actual 4.25%" (hold 표시)
- Mar 19 FOMC: "Estimate 4.00% | Actual 3.75%" (50bp surprise cut 표시)
- April 30+ : actual=null 유지 (미개최 회의 정상)
