## 발견된 버그

### Bug: fear-greed force=1 캐시 우회 미작동 (HIGH)

**문제:**
`/api/fear-greed?force=1` 으로 cron/update-all 에서 강제 새로고침 호출.
그러나 `GET()` 함수가 request 파라미터를 받지 않음:
```typescript
// 수정 전:
export async function GET() {  // request 파라미터 없음 → URL 파싱 불가
  ...
  // buildEntry 내부 Redis 체크:
  if (redis) { ... if (cached) return cached; }  // force=1 무시
```

결과:
- cron이 `?force=1`으로 호출해도 Redis에 캐시된 값이 있으면 항상 stale 반환
- CNN F&G 실제값이 변해도 4h TTL이 만료될 때까지 갱신 불가
- 2026-04-22 stale 사건 (CNN score 67.57 vs 우리 70) 재발 가능성

**수정 (commit 7918beb):**
```typescript
// 수정 후:
export async function GET(request: Request) {
  const { searchParams } = new URL(request.url);
  const force = searchParams.get('force') === '1';
  ...
  // buildEntry에 force 파라미터 전달:
  buildEntry(id, ticker, ..., redis, useCNN, force)

  // buildEntry 내부:
  async function buildEntry(..., force = false,) {
    if (redis && !force) {  // force=1 시 캐시 스킵
      try { const cached = ...; if (cached) return cached; }
    }
  }
```
메모리 캐시도 `!force` 조건 추가 (no-Redis 경로 일관성).

**영향 범위:**
- `/api/cron/update-all`: cron이 3회/일 force=1로 호출 → 이제 실제 CNN/Yahoo 최신값 fetch ✓
- verify-metrics 정확도 향상 (cron 직후 4h 내 probe 시 최신값 반환)
