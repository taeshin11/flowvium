### Bug 2: YTD vs 단일분기 값 중복 시 잘못된 선택 (MEDIUM)

**문제:**
SEC XBRL는 같은 10-Q 파일에 YTD context와 기간(period) context를 모두 포함 가능.
예: JPM Q3 2025 = `end=2025-09-30, fp=Q3` 항목이 두 개:
- YTD context: $136.6B (1월~9월)
- 단일 분기 context: $46.4B (7월~9월)

기존 dedup 로직: `if (!existing || e.end > existing.end)` → 동일 end 시 첫 번째 값 보존
→ 반복 순서에 따라 단일분기 값($46.4B)이 YTD 값을 덮어쓸 수 있음

YTD를 단일분기로 오인하면:
- Q4 = Annual - $46.4B = $136B (비정상 대형 값)
- 또는 Q3 = $46.4B - YTD_Q2($90.2B) = -$43.8B → filter(val>0)로 걸러지나 Q4도 오염

**수정 (commit fda673a):**
```typescript
// BEFORE:
if (!existing || e.end > existing.end) ytdMap.set(key, e);

// AFTER: 동일 end 시 더 큰 값 선택 (YTD > 단일분기 항상 성립)
if (!existing || e.end > existing.end || (e.end === existing.end && e.val > existing.val)) {
  ytdMap.set(key, e);
}
```

**검증:**
- Q3 YTD: $136.6B vs 단일: $46.4B → 136.6 > 46.4 → 올바른 YTD 선택 ✓
- Q2 YTD: $90.2B vs 단일: $44.9B → 90.2 > 44.9 → 올바른 YTD 선택 ✓
