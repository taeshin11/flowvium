### 2. 발견된 버그: 70b non-429 HTTP 오류 시 8b 폴백 스킵 (MEDIUM)

**문제:**
기존 코드: `if (primary.status === 429)` → 8b 폴백 조건이 429 전용.
70b가 500/503 (일시적 서버 오류) 반환 시 8b 시도 없이 Qwen으로 escalation.
실제 케이스:
- GROQ 서버 일시 과부하 → 70b 503 반환 → 8b는 정상인데 Qwen 호출 → OpenRouter 지연
- 70b 특정 엔드포인트 500 → 8b는 별도 리소스풀 → 8b 정상 가능성 있음

**수정 (commit b454929):**
```typescript
// BEFORE:
if (primary.status === 429) {
  logger.info(tag, 'groq_429_fallback_8b', ...)

// AFTER:
if (primary.status !== null && primary.status !== 200) {
  logger.info(tag, 'groq_http_fallback_8b', { note: `70b status=${primary.status}, ...` })
```

**주의: 타임아웃 시 스킵 유지** (`status === null` = 네트워크 오류/타임아웃):
- 70b timeout = 20s → 8b도 동일 GROQ 네트워크 문제일 가능성 → 스킵 ✓
- HTTP status 반환 = 서버 응답 있음 → 8b 별도 리소스풀 가능 → 시도 ✓

### 3. TPD Redis guard 정확도 개선 (MEDIUM)

**문제:**
기존: `if (fallback.status === 429 && fallback.tpdExhausted)` → 70b가 500이어도 8b TPD 소진 시 글로벌 GROQ guard 쓰기.
→ 70b 일시 과부하(500) + 8b TPD 소진 케이스에서 70b를 잘못 차단.

**수정 (commit b454929):**
```typescript
// BEFORE:
if (fallback.status === 429 && fallback.tpdExhausted)

// AFTER:
if (fallback.status === 429 && fallback.tpdExhausted && primary.tpdExhausted)
```
양쪽 모두 TPD 소진일 때만 Redis guard 쓰기 → guard 오염 방지 ✓
