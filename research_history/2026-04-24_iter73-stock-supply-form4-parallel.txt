### 발견 경위
Explore agent scan: stock-supply/route.ts의 fetchInsiderTransactions 내 Form 4 XML fetch가
6개 순차 await (~1s/건) → ~5-8s 총 소요.
이미 Promise.allSettled가 volResult/insidersResult 병렬화에 쓰이지만,
insiders 내부 루프는 여전히 sequential.

### 구현 (`src/app/api/stock-supply/route.ts`)

#### Before (lines 102-158): sequential for-loop
```typescript
const txns: InsiderTx[] = [];
const cikNum = cik.replace(/^0+/, '');
for (const f of form4s.slice(0, 6)) {
  try {
    // await fetch(xmlUrl, ...) — each ~1s
    ...
  } catch { continue; }
}
return txns.sort(...);
```

#### After: Promise.allSettled parallel fetch
```typescript
const cikNum = cik.replace(/^0+/, '');
const xmlResults = await Promise.allSettled(
  form4s.slice(0, 6).map(async (f) => {
    const xmlRes = await fetch(xmlUrl, { signal: AbortSignal.timeout(5000), ... });
    if (!xmlRes.ok) throw new Error(`HTTP ${xmlRes.status}`);
    // parse XML → return fileTxns[]
  })
);
const txns = xmlResults
  .filter((r): r is PromiseFulfilledResult<InsiderTx[]> => r.status === 'fulfilled')
  .flatMap(r => r.value);
return txns.sort((a, b) => b.date.localeCompare(a.date));
```

#### 주요 변경 사항
- sequential for-loop → Promise.allSettled map
- continue (network error) → throw new Error() → Promise.allSettled handles
- 각 filing의 fileTxns[] → flatMap으로 병합
- 정렬 보존 (sort by date descending)
- AbortSignal.timeout(5000) 유지 (per-request timeout)

### 성능 영향
| 케이스 | Before | After |
