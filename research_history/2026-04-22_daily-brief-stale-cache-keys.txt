## 이번 iteration — daily-brief 품질 치명적 버그 발견
### 증상
```
GET /api/daily-brief?tf=4w → source:"GROQ-llama-3.3-70b", cached:false
  market.content = "시장"          ← 단어 한 개
  capital.content = "자금"         ← 단어 한 개
  bullets = undefined
```

GROQ 호출은 정상이었음. 따라서 AI 자체는 생성 성공. 문제는 **프롬프트가 비어있음**.

### 조사
gatherTabContext(redis) 에서 Redis 키 조회 5개를 대조:
- daily-brief가 읽음 → 실제 write 키
- `flowvium:capital-flows:v4:twelve` → 현행 `v5:twelve` ❌ (항상 null)
- `flowvium:capital-flows:v4:yahoo`  → 현행 `v5:yahoo`  ❌ (항상 null)
- `flowvium:capital-flows:v4:none`   → 현행 키 없음       ❌ (항상 null)
- `flowvium:fg:v3:SPY`               → 현행 `v5:SPY`    ❌ (항상 null)
- 나머지(heatmap:v5:US, fedwatch:v1, macro-indicators:v4, credit-balance:v2, insider-trades:v1, ...) 모두 매칭 ✓

### 결과 영향
- ctx.capital = null → summariseCapital 빈 문자열 반환
- ctx.fearGreed = null → summariseFearGreed 빈 문자열
- 2개 주요 데이터 소스가 프롬프트에서 빠짐 → AI가 문맥 없이 "시장","자금" 같은 placeholder 생성
- 이게 얼마나 오래 지속됐는지는 git log 필요 (capital-flows v4→v5, fg v3→v5 마이그레이션 이후 줄곧)
