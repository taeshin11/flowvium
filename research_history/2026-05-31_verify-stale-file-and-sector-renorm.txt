## 발견 1 (최고 임팩트) — verify 가 며칠째 stale 보고서를 검증
- 증상: `npm run verify` 의 verify-latest-report 가 5/30 morning 보고서(14결함)를 검증.
  실제 최신은 5/31 evening(3결함). 검증이 잘못된 파일을 보고 있었음 = 사각지대.
- root cause A: `scripts/verify-report.mjs:192` default 파일이
  `'reports/report-2026-05-30-morning-ko.json'` 하드코딩 → 인자 없이 호출되면 항상 그 파일.
- root cause B (진짜 원흉): `scripts/verify-all.mjs:91`
  `readdirSync(...).filter(...).sort().slice(-1)` — lexicographic 정렬로
  afternoon < evening < morning → "morning" 을 최신으로 오선택. evening 무시.
  verify-all 이 이 잘못된 파일을 명시적 인자로 verify-report 에 넘겨 default fix 무력화.
- fix:
  - verify-report.mjs: `pickLatestReport()` 추가 (파일명 날짜 + 세션 rank{morning<afternoon<evening}
    정렬) + export. default 를 pickLatestReport() 로 교체 + "검증 대상:" 로그.
  - verify-all.mjs: line 91 자체 정렬 제거, `import { pickLatestReport }` 단일 source of truth.
    미사용 readdirSync import 제거.
- 검증: pickLatestReport('reports') → report-2026-05-31-evening-ko.json (양쪽 동일).
  verify-all 의 verify-latest-report err 7 → 4 (이제 evening 정확 검증).
