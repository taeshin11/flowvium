FIX 1 — short-interest/route.ts: O(n²) liveSignals.find() 제거
  문제: 500+ 신호 루프 내에서 각 iteration마다 liveSignals.find() 재탐색 → O(n²)
  해결: instFilingDateMap 추가 → O(1) 날짜 비교로 대체 → O(n)
  예상 절감: 50~200ms (신호 데이터 크기에 따라)
  Commit: 63da7dc

FIX 2 — short-interest/route.ts: Redis 13f-signals 순차 → 병렬
  문제: FINRA+Finnhub 병렬 fetch 완료 후 Redis 순차 읽기
  해결: Promise.allSettled에 Redis 읽기 합류 (3개 동시 실행)
  예상 절감: Redis RTT ~50ms 중첩
  Commit: 63da7dc

FIX 3 — flow-analysis/route.ts: 순차 Redis 2회 → 병렬
  문제: 캐시 miss 경로에서 fresh+stale Redis 키를 순차 읽기 (~300ms+300ms)
  해결: Promise.allSettled([fresh, stale]) 병렬 읽기
  예상 절감: 200~300ms (miss 경로)
  Commit: 63da7dc

FIX 4 — market-heatmap/route.ts: triple reduce → single loop
  문제: 11개 섹터 × (filter + reduce×2) = 33 array pass
  해결: 섹터별 단일 for loop에서 totalMC + weightedSum + weightedDenom 동시 계산
  예상 절감: 5~20ms (배열 크기 작음)
  Commit: 63da7dc

FIX 5 — cron/update-all/route.ts: N+1 stock-supply → Promise.allSettled
  문제: TOP_TICKERS 10개를 for 루프에서 순차 fetch (fire-and-forget이지만 불필요한 순차)
  해결: Promise.allSettled(TOP_TICKERS.map(...)) 병렬 실행
  예상 절감: 크론 총 실행 시간 단축 (10 × EDGAR RTT 중첩)
  Commit: 5e0441b
