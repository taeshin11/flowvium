## 진단 경로 (debug=1 probe로 확정)
직전 iteration v4→v5 key 수정이 효과 없었음. 원인 분석 위해 /api/daily-brief?debug=1 임시 프로브 추가 → 응답:
```
"redisConfigured": false,
"ctxPopulated": { heatmap: false, capital: false, fearGreed: false, ... signalsCount: 56 }
```

**결정적 발견**: createRedis() 가 Vercel 환경에서 null 반환 = UPSTASH_REDIS_REST_URL/TOKEN 미설정.
- verify-metrics cache items 프로브: 0개 (createRedis null → verifyRedisCaches 스킵)
- 즉 **프로젝트 전체 Redis 없이 운영 중**
- 이는 이전 세션 2026-04-22_verify-metrics-race-and-daily-brief 로그에서도 언급됨 (UPSTASH 병목)

### 왜 capital-flows, fear-greed는 정상 응답했는가?
- Redis 없어도 route 내부에서 fresh compute → 반환 정상
- 2회차 호출 빠른 응답은 Vercel Next.js fetch 캐시 / 함수 인스턴스 웜업 효과 (Redis 아님)

### 왜 daily-brief는 shallow 했는가?
gatherTabContext:
```ts
if (!redis) return ctx;   // ← empty ctx early return
```
→ 프롬프트 [CapitalFlows] n/a, [Fear&Greed] n/a ... → AI 가 "미제공" placeholder 생성
