BUG: daily-brief.ts gatherTabContext() news cascade 항상 빈 배열

  try { return await redis.lrange(`flowvium:news-cascade:v1:list:${today}`, 0, 5); }
  catch { return [] as string[]; }

- redis.lrange()는 Redis LIST 타입 명령어
- 하지만 news-cascade/route.ts는 listKey()를 redis.set()으로 저장 (JSON string)
- JSON string에 lrange 호출 → Redis WRONGTYPE 에러 → catch → 빈 배열 []
- 결과: ctx.cascade는 항상 [] → AI 프롬프트에 뉴스 컨텍스트 0개
- 동일 버그가 secondary lookup도 dead code로 만들었음:
  cascadeIds.slice(0, 5).map(id => safeGet(redis, `...article:${id}`))
  → id가 문자열이 아니라 article 객체여야 했는데, 어차피 cascadeIds는 항상 []
