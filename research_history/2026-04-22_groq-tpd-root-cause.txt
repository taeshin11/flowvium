```json
  "durationMs": 79

```

## 완전한 그림
GROQ 무료 티어는 **2개 독립 한도**:
1. RPD (Requests Per Day): 14,400 건
2. **TPD (Tokens Per Day): 100,000 토큰** ← 훨씬 빨리 소진

이번 세션에서 iteration 많이 돌며 daily-brief(1800 max_tokens × 수십 회) + flow-analysis + translate + news-cascade 등이 토큰 누적 → 99,590/100,000 소진 → 429.

### 왜 verify-metrics GROQ probe 는 통과했는가?
max_tokens=1 + content="ok" → 실제 소비 ~5 토큰/호출. TPD 한도에 거의 영향 없음.
**즉 probe 자체는 지나치게 싸서 quota 소진을 탐지 못 함** — 기존 probe의 맹점.

## 수정 (src/app/api/cron/verify-metrics/route.ts)
verifyAIProviders 의 GROQ probe 확장:
- 응답 헤더 x-ratelimit-* 에서 TPD/RPD limit+remaining 읽음
- tpdPctUsed >= 90% 면 status='degraded' (사전 경보)
- value 에 "TPD 99.6% used (410/100000 left)" 표시
- details 에 tpdLimit/tpdRemaining/rpdLimit/rpdRemaining 기록

효과:
- 토큰 소진 임박 시 30분 크론마다 감지 → admin/logs 에서 즉시 경고
- 완전 소진(429) 은 기존대로 degraded 분류 + 'quota exhausted' 문구
